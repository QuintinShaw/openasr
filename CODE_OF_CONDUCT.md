# Code of Conduct

OpenASR is a local-first speech-to-text project. The project expects contributors and maintainers to keep collaboration respectful, technical, and focused on improving the software and documentation.

## Expected Behavior

- Be respectful and direct.
- Assume good intent, but correct inaccurate technical claims with evidence.
- Keep discussions scoped to OpenASR behavior, docs, tests, privacy, provenance, and user impact.
- Use public issues only for material that is safe to share publicly.
- Redact private audio, credentials, secrets, local paths, and private model/runtime artifacts when needed.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private audio, credentials, secrets, or private artifact links without consent.
- Pressuring maintainers to publish unverified downloads, fake hashes, private runtime assets, or unsupported release claims.
- Repeatedly derailing issues or reviews after maintainers have asked to return to scope.

## Reporting

Report conduct concerns to the repository owner through GitHub. For security issues, follow [SECURITY.md](SECURITY.md).

Maintainers may edit, hide, lock, or remove comments and issues that violate this code of conduct. Maintainers may also limit participation for repeated or severe violations.
